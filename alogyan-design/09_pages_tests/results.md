# Results

**References:** Google Classroom (grades view).

## 1. Layout
Detail view. Top section shows aggregate statistics (Avg, High, Low score). Main body is the spreadsheet-style marks entry.

## 2. Components
- **Summary Banner:** "Unit 1 Physics - Avg Score: 76%".
- **Spreadsheet Grid:** Roster list with inputs.

## 3. Tables
Data table for entering marks. Student Name (readonly) | Score (input) | Feedback (input).

## 4. Forms
In-line table editing form for high-speed offline mark transcription.

## 5. Buttons
"Publish Marks" (Primary Right).

## 6. UX Flow
Teacher selects a past test -> Clicks "Enter Marks" -> Types heavily using Tab key -> Submits form -> Parents get WhatsApp updates.

## 7. Mobile Layout
Not recommended for spreadsheet view. Degrades to a vertical card swiping experience (One student per screen) for mobile entry.

## 8. Empty States
"No students found in this batch to score."

## 9. Success States
"Marks accurately recorded and published. Notifications sent."

## 10. Error States
"Score cannot exceed maximum possible marks (50)." inline alert.
