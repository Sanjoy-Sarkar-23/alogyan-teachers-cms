# Typography

**References:** Modern SaaS platforms utilizing standard web-safe or highly legible sans-serifs (like Inter in Stripe, or Roboto in Google Classroom).

## 1. Layout
Hierarchy is maintained strictly. H1 for page titles, H2 for sections, H3 for cards, P for body text.

## 2. Components
- **Primary Font:** `Inter` or `Google Sans`. 
- **Sizes:** Micro (12px), Small (14px), Base (16px), LG (18px), XL (22px), 2XL (28px).

## 3. Tables
Table headers are `14px Semi-Bold`. Data is `14px Regular`.

## 4. Forms
Form labels are `14px Medium`. Form inputs are `16px Regular`.

## 5. Buttons
Action text is `16px Medium` to ensure legibility.

## 6. UX Flow
Dark grey `#212121` is used for primary reading to reduce eye strain compared to pure `#000000`.

## 7. Mobile Layout
Base font size `16px` remains standard on mobile to prevent automatic zooming by iOS Safari when interacting with inputs.

## 8. Empty States
"Empty" messages use a slightly lighter grey (`#757575`) and a larger font (`18px Regular`) for a soft, approachable tone.

## 9. Success States
Success messages in bold (`Medium`/`Semi-Bold`) to reassure the user.

## 10. Error States
Error states not only use Danger Red but also an accompanying icon (⚠) since color alone is insufficient for accessibility.
