# Colors

**References:** Google Classroom (Education theme), Razorpay (Clear status colors).

## 1. Layout
Colors define the visual hierarchy. Primary Red `#D32F2F` draws the eye to the main action.

## 2. Components
- **Primary:** `#D32F2F` (Alogyan Red)
- **Success:** `#2E7D32` (Paid, Present, Active)
- **Warning:** `#F57C00` (Due, Pending, Late)
- **Danger:** `#C62828` (Absent, Overdue, Remove)

## 3. Tables
Table headers are tinted with `#F5F5F5`. Hovering over a row changes its background to `#FAFAFA`.

## 4. Forms
Input borders default to `#E0E0E0`, shifting to Primary Red on `:focus`.

## 5. Buttons
Colored based on severity: Normal actions are Primary Red. Destructive are Danger Red. Success/Verify actions are Success Green.

## 6. UX Flow
Semantic coloring ensures users immediately understand context without reading text (e.g., a green badge always means something good).

## 7. Mobile Layout
High contrast is maintained for outdoor phone usage (e.g., dark text `#212121` on white).

## 8. Empty States
Empty states use muted, disabled colors (`#BDBDBD`) so they don't demand immediate attention like active tasks.

## 9. Success States
Forms flash a subtle `#E8F5E9` (Light Green) background upon successful submission.

## 10. Error States
Danger Red `#C62828` is used for validation text and input borders on error.
