# Spacing System

**References:** Notion (breathing room, relaxed data density).

## 1. Layout
Based on an **8px grid system**. Margins and paddings are always multiples of 8 (8, 16, 24, 32, 40, 48, 64).

## 2. Components
Inner component padding (e.g., inside an input or button) uses `8px` vertical and `16px` horizontal.

## 3. Tables
Table cells use `12px` vertical and `16px` horizontal padding for a relaxed, readable density.

## 4. Forms
Spacing between form fields vertically is `16px`. Spacing between distinct form sections is `32px`.

## 5. Buttons
Horizontal gap between adjacent buttons (e.g., "Cancel" and "Save") is `16px`.

## 6. UX Flow
Consistent spacing creates a predictable, vertical rhythm, reducing cognitive load when scanning down a page.

## 7. Mobile Layout
Global page layout padding drops from Desktop (`40px`) to Tablet (`24px`) to Mobile (`16px`).

## 8. Empty States
Empty states feature large surrounding negative space (e.g., `64px` padding) to isolate the call-to-action in the center.

## 9. Success States
Toasts and snackbars hover `24px` above the bottom edge of the screen.

## 10. Error States
Error messages below form inputs have a `4px` top margin, keeping them tightly grouped with their parent input.
